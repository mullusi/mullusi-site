Security.

This page summarizes the controls that apply to the static Mullusi public website and explains how to report a suspected issue.

These controls apply to the public static site. Runtime gateway and operator surfaces are not presented as live public services yet.

Scope Public website routes only.

Reports should focus on mullusi.com pages, static assets, public JSON, security headers, and published contact paths.

Report Use disclosure or support.

Send suspected issues through the responsible disclosure page or support mailbox with impact and safe reproduction details.

Triage Classify, repair, and record.

Mullusi reviews the report, separates website issues from staged services, and records the response boundary.

Limit Reserved services are not targets.

API, dashboard, sandbox, metrics, operator, and private product systems are not open for public testing.

Browser policy

The Cloudflare Pages header set includes Content Security Policy, HSTS, frame blocking, nosniff, strict-origin referrer policy, and a permissions policy that disables powerful browser APIs.

Script boundary

Executable scripts are self-hosted and external. The only inline script currently allowed is the hash-pinned Organization/WebSite JSON-LD block on the homepage.

Route boundary

Reserved subdomains are not linked as live endpoints. Visitors are routed to published static pages, status, contact, or proof boundary pages.

Data boundary

Public JSON files are static, content-hashed, and validated before deployment. Runtime proof stamps are not claimed until witness endpoints close.
